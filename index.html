Getting Stockr online

5/7/16

It was more of a struggle getting Stockr running online than I expected, I think because I've never launched a Nodejs app and they have some funky requirements for getting the Node server to continue running remotely even when I'm not logged into my production server and telling it to run. I found a npm package call Forever that has done the trick for now, though it's not supposed to be that great a method. I'm still trying to figure out if it will work fine for my small app's needs, and if Forever's limitations are more something to worry about for enterprise apps.

I haven't had a chance yet to get Stockr running at a domain name yet, but you can access it at it's IP address here: http://104.236.243.250:8080/?#/

This app is still pretty fragile, and there are some definite kinks to work out. Please use the guidelines below to get it working for the time being :p

  • Use Chrome to view the app.
  • The first time you click the form's "Compare button," data doesn't yet populate to the page. I think what's going on is that the data has been pulled from the APIs, but my app has gotten the updated information in time to print it. I'm working on getting the results to print to the page as Stockr receives them, but for the time being clicking the "Compare" button twice will allow the results to show

Those are the main things to keep in mind, I found the others were quick fixes as I was writing this post. You can find my code's repository here: https://github.com/ktbee/stockr

Solving scope issue with Angular

4/18/16

These last two weeks I found that incorporating a way for my users to send information to my app was a little trickier than I had anticipated. I was able to have search parameters submitted through the frontend form -> to the app's API -> to the stock market API -> and back to the browser's web tools console. However, I couldn't get the information retreived through this process to display on the web page and I didn't receive an error message. Eventually I found out that I was accidentally creating two scopes, and that my page was outside the scope of where my stock information could print.

Now that I can finally print stock prices to my page, I'm working on incorporating D3 into my Angular app. I found this helpful blog post on how to add simple bar graphs: http://briantford.com/blog/angular-d3

My goal for this week is to be able to create bar graphs on the frontend using D3 with some mock data. My stretch goal is to refactor how I make handle data from my API requests to TradeKing so that I can display it with D3 more easily.
