Creating a story map for my stock market app

2/25/16

In developing my app for analyzing the stock market, I cam up with these answers to some of Shaunalynn and Alec's questions:

  • What is the project's mission?
    Allow the user to find connections between changes in stock market and their selected API
  • Who is its audience, and what do they value?
    Audience is myself. I value finding unusual connections between two different events or concepts that is backed up with data. A secondary audience may be other researchers looking to find a correlation they can research further.
  • How will you know if it is successful?
    It will be successful if I can generate charts or word maps based on user's parameters for how to compare an API to the stock market one.

Generative art and data visualization

2/18/16

I had a great chat with Molly about developing some project ideas. Afterwards, I found that my interests fall into two main categories:

  • Generative art possibilities:
    • Create tree ring growth pattern to use in current furniture project
    • Code-ify bed bug DNA sequence into a knitable pattern for a blanket
    • Create a random color chart generator for knitting patterns that takes in colors and sizes and generates unique color patterns each time
  • Data visualizations:
    • For a final project, create an app that incorporates a stock market API with other ones for comparison (like Sunlight Foundation, weather, New York Times) to see if there are any trends to certain stock market changes
    • Make more whimsical visualizations to answer questions like, does the length of pockets created in women's pants for a certain time period correspond to average purse sizes at that time?

My plan now is to read about some resources on how patterns form in nature and start doing D3.js tutorials to try my hand at some straight forward data visualizations. I will practice using D3 on open data sets I find interesting, like ones from Socrata
